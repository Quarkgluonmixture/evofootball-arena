## 1. 灵魂:涌现,不是我们设定 (EMERGENCE — the soul)

**用户原话（2026-07-14）:**
> 真正的应该是自己涌现各种风格、技巧、二过一、战术等等,而非我们去设置。
> 这才是我更希望的 evo 的含义。

> 不要写死预设,让他发挥才智。

**铁律:**
- 风格 / 技巧 / 二过一 / 战术 / 跑位 **从进化 + 选择里长出来**,不是我们手写。
  手写"边锋做 X"、给某个角色调一个必然触发的结果,**违背灵魂**。
- **底座只提供"允许 X"的能力,不替球队"定 X"**（用户 2026-07-19：
  "这些都是'底座现在允许 x'而不是'我们帮球队定了 x'"）。给能力 + 基因门控,
  让**选择**决定谁用、用多好;用不好的被淘汰,天经地义。
- **诊断问题 = 诊断进化问题,不是设计问题**:缺失的好行为通常是**局部最优**
  ——它在当前底座里不划算,所以进化正确地没选它。修法 = 把底座改成让好行为
  **划算**(gene-driven、无角色偏置),退掉补偿性的手调,然后**跑演化并观察**。
- 诚实张力:不是一切都能涌现——**物理 + 基因维度**是我们手搭的底座;涌现的是
  **战术层**;底座的职责是让基因**有意义**。

## 2. 观赏性与手感 (WATCHABILITY & FEEL)

**用户最烦的:中场乱抢（2026-07-18/19，反复强调）。** 原话——不喜欢的三个原因:
> 他没有战术,没有观赏性,不喜欢所有人一起挤在一起。

具体现象（用户）:
> 绝大多数都是中场好多人一起挤,然后球乱飞突然变成单刀的。

**所以要:**
- **不要所有人挤在中路**;不要"乱抢→球乱飞→突然单刀"这种便宜、无战术的进球。
- **战术要看得出来**（"可视化优秀"）——不同打法在场上肉眼可辨,不是一团混沌。
- **流畅 > 打断**:play-feel 优先,别让机制补丁把比赛切碎（memory: fluency > interruptions）。
- **默认干净**（用户 2026-07-19："把默认调试图层都关闭"）:第一眼看到的是**足球**,
  不是调试图层。调试层仍可在面板里开。
- 用户**真的在玩**(含**手机**,≤390–640px)——观感与操作以真实游玩为准;
  headless 截图对像素不可信,信用户的浏览器。

⚠ 任何"让 X 涌现"的机制,如果把比赛变得**更乱抢 / 更难看 / 战术更糊**,就是
**失败**——即使它在别的指标上赢（2026-07-19 缩场教训:naive 缩场让乱抢恶化 →
不可接受;必须协调地做，观感是硬门槛）。

## 3. 贴近现实 (REALISM — reference real football)

**核心信条:如果现实足球不会这样,我们的底座就不该这样。** 用户 2026-07-19 的挑战:
> 如果真的最优解是中路乱挤,为什么现实 6v6 不会这样?世界 6v6 比赛冠军
> 都是那种把人堆到中间得了?

对照现实来诊断能力值/属性/机制（用户:对照实况/FIFA/FM）。

### ⭐ 3.0 "现实机制" = 整场游戏,不只是防守（用户 2026-07-19）

> 现实机制还有那些远射,挑射,过掉门将,以及边路盘带,内切,到中路一般怎么做,
> 回撤,一脚回垫等等……现实机制指的不只是防守,还有进攻,build up,套路等等,
> 这些都应该从一个优秀底座长出来。

**警戒:别把"贴近现实"窄化成防守。** 一个优秀底座要让**整场的真实机制**都能涌现:
**进攻、build-up、套路,和防守**同等重要。下面列的动作**都是举例、不是穷举**,
真正的标准是"**现实足球里存在的机制,在我们的底座里都该是可能的、gene/attr 驱动的,
让合适的球队/球员进化去用**"(§1 灵魂:给能力,不手写)。已知要覆盖的层面:

- **射门 repertoire**:远射、挑射（吊射)、过掉门将（趟过/推空当)、near/far post、
  抢点、二次射门…… 由 finishing/技术/决策基因驱动,不是一种射门。
- **持球/突破 repertoire**:边路盘带、内切、外线下底、**到中路后一般怎么处理**（分球/
  做球/起脚/继续带)、变向摆脱、护球…… 边路武器库的细节见 §3.1。
- **接应/做球/链接**:**回撤接球**(宽度+长度都可回撤)、**一脚回垫/做球**（lay-off)、
  三角配合、二过一、撞墙、第三人跑动…… 传控队靠这些推进。
- **build-up（后场组织)**:门将/后场出球的真实选择,由后往前的层层推进,
  被逼抢时的解压出球,长传转移…… 不是无脑大脚或一条道。
- **套路（set-plays + 固定配合)**:角球/任意球/界外球的真实套路,以及**运动战里的
  固定配合**（overload-to-isolate、肋部三角、下底倒三角等)——都应能作为可选择的
  打法涌现,而非写死。

### 3.1 已确立的现实锚点（本会话深挖:防守 + 边路武器库 —— 现实机制的一部分）

这些是 2026-07-19 这轮已经量化确立的锚点(防守两范式 + 中/边不对称 + 边路武器库);
它们是 §3.0 全场机制里被先钻研的一角,不是全部:

- **防守两大范式的权衡,参考现实**（用户 2026-07-19："过一个还有另一个 vs 盯人
  的权衡,应该参考现实"）:
  - **区域防守 = "过一个还有另一个"**（分层补防,中路盘不穿;被缝隙/超载/无人盯的插上破）。
  - **盯人**（没有自由人;但过了你的人就没人补 + 可被跑动扯出空当）。
  - → 石头剪刀布:盘带克盯人、宽度/配合克区域,各有克星,**从 scheme 基因涌现**。
- **中路 vs 边路的空间不对称,要从底座长出来**（用户 2026-07-19）:
  > 中路盘带很容易被很多人夹击,边路往往有空间,人数优势。这一点需要底座
  > 本身就能让他体现。
- **边路是一整套武器库,都要有底座支撑**（用户 2026-07-19）:
  高空传中 / 低倒三角 / **还没到底线就给肋部前插的人（早球）** / **边路三角配合
  （二过一、做球、打身后 —— 传控队常做）**。
- **边锋宽度应该是能变的**——回撤接球时**宽度 + 长度都可能回撤**（朝球靠,拿稳再拉开进攻）。
- **夹击/补防是球队能自由选择的,不是我们硬规定的**——有的队不夹击,结果被打穿,活该。
- 细节真实:解围一般**有方向**（不是乱踢）;等等。

**故意的街机式偏离——不要"修"回去**（deliberate arcade deviations, memory）:
越位 → 门球（不是任意球）、禁区内持球门将解围清空、门球适用越位……这些是**有意**的
简化/取舍,不是 bug。改动前先分辨是"违背现实的缺陷"还是"故意的街机取舍"。

## 4. 涌现的多样性 (EMERGENT DIVERSITY)

- 战术 / 风格 / 技巧 / 二过一 **自己长出来 + 互相竞争 + 进化**（见 §1）。
- **多样性需要约束**（用户 2026-07-14）:没有资源上限,进化会把每项属性拉满 →
  "什么都强"胜出 → 风格不分化。**squad 点数预算**（升一项要降一项）逼出取舍 →
  不同原型涌现。✅ 已上线（SQUAD_BUDGET）。
- **进化结果要更快 + 更显眼**（用户 2026-07-14）:分化不应到 20–50 赛季才看得出;
  要 style 铭牌 + 可视化让它早早肉眼可见。目标:≤8–10 赛季可见聚类。
- **每个个体都有赢来的身份**:每个球员、每个教练、每个赛季/杯赛都有独特闪光点
  与打法,且**都在 evo**。铭牌/时代/叙事**从数据发现**,不是预设分类。

## 5. 会进化的世界 (THE EVOLVING WORLD — 4 层)

**顺序 = 杠杆**（binding）:引擎（有多样性）→ 可视化（多样性被看见）→ 世界（多样性有
面孔和故事）→ 底座加厚。

- **引擎**:进化真正产出多样性（属性扩展 + 预算 + results-dominant 适应度）。
- **可视化**:style-space 地图、分化曲线、数据驱动铭牌——让进化**看得见**。
- **世界**:**教练**（战术基因的化身 + 通过流动传播哲学的 memetic 通道）、**转会/自由市场**
  （第二条基因通道）、**赛季/杯赛编年史**（完全从记录派生,时代名从数据聚类涌现）。
- **底座加厚**（长期,拉动式）:每项都要过"它给基因一个**新的收益面**吗?"的测试
  ——轮换/替补、状态/士气、伤病、主场效应、有记忆的解说。
- **元规则贯穿每一层**:教练哲学是进化基因,不是预设原型;叙事层**只读**记录,永不反哺 sim。

## 6. 工作纪律 (HOW WE WORK — 也是用户明确要的)

- **一次一根杠杆 + probe-first + A/B 才提交**:每 phase 一个杠杆,先探针量化,
  对照上一 tag A/B,过全门（vitest、visual-debug×2、calibrate、goals-warming、
  fingerprint 重刷),再提交/打 tag/推。
- **诚实回退**:测不过就回退,如实报告数字;别把不 work 的东西硬上。
- **A/B 只决定"这一版 ship 还是 revert",不决定"这个方向是否死"**（revert→reframe,
  用户 2026-07-18）:被回退的尝试可拆成"安全 UI 半"+"探针门控的行为半",
  以蓝图要求进入未来 phase。
- **play 报告驱动 phase**:用户玩 → 反馈 → 排期。
- 记忆写 durable-only;live 状态进账本（CHECKPOINT/ROADMAP/BLUEPRINT）。

## 7. 怎么用这份金标准 (HOW TO USE THIS)

1. **每个改动先问:它服务 §0–§5 的哪一条?违反哪一条?** 违反愿景的,愿景赢——
   哪怕它让某个指标好看。
2. **"让某某涌现"的杠杆,必须同时通过 §2 观赏性硬门槛**（不能更乱抢/更难看/战术更糊）
   和 §1 涌现纪律（不能手写行为、只能给 gene-gated 能力）和 §3 现实锚点。
3. 大结构改动（如缩小球场以匹配 6v6 密度）**只有在能同时满足 §2 + §3 + §1** 时才值得投;
   probe 先行,拿本文件当验收标准。
4. 有新的"我希望"就并进来,并在相关账本里回指本文件。
