# Emergent Re-entry Ledger

Status: **active migration authority for reverted or scripted football patterns.**

Date: 2026-07-21

## 1. Binding principle

> A reverted implementation is not a rejected football phenomenon.

The user observed the important example: cut-inside used to be hand-written, but
under the newer engine it can appear as a natural carry trajectory. That is the
correct re-entry shape. We may reopen an old direction only when a causally new,
generic substrate makes it possible without naming or rewarding the pattern in
the live decision path.

These are different claims:

```text
old implementation failed
!=
football phenomenon is impossible or unwanted

pattern observed after the fact
!=
pattern selected by a named live action
```

## 2. Re-entry contract

Every returning direction must satisfy all of these:

1. **Name the missing generic fact or primitive.** Examples: reachable candidate
   point, access lane, teammate task occupancy, receive margin, pass-to-space or
   control-to-direction. `CutInside` and `Overlap` are not primitives.
2. **Keep football pattern names observational.** Pattern labels may appear in
   probes, telemetry, replays and narration only. They cannot be genes, role
   bonuses, action enums, target generators or hidden utility bonuses.
3. **No role privilege.** A winger may produce more cut-inside trajectories
   because of position, body state, attributes and evolved preferences—not because
   `role === WG` unlocks one.
4. **Preserve the accepted live implementation as the baseline.** A generic
   replacement retires a named script only after mechanism, payoff, selection and
   watchability gates pass.
5. **Reuse history, not authority.** Old failures constrain counterfactual tests;
   they do not waive the new slice's probe/A-B/ship-or-revert contract.
6. **One causal lever per slice.** Representation, live consumption, team
   coordination and gene selection receive separate gates.

## 4. Current first cut: off-ball candidate/affordance representation

The next mainline slice is deliberately pre-behavioural:

```text
observer-specific snapshot
+ player reach profile
+ generic, role-neutral reachable points
→ unscored off-ball affordance vectors
```

It may expose facts such as self arrival, opponent arrival, teammate occupancy,
carrier lane clearance, progression, field legality and offside margin. It may not:

* select a point in live AI;
* replace `supportSpot` yet;
* add a tactic label or role bonus;
* add a gene or scalar utility;
* claim payoff from candidate coverage alone.

Passing this slice proves only that the missing choice space and its causal facts
exist. Live selection, team coordination and script retirement each require their
own later contract.
