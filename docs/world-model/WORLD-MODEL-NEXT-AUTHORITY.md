# World-Model Next Authority after D-COVER / D-LANE

Status: **OPTION A MECHANISM GATE COMPLETE — FAIL; selection/payoff remain closed.**

Date: 2026-07-22

## 1. Plain conclusion

The project does not mainly lack another distance, lane or timer. It has many
useful facts, but it still lacks a trustworthy **temporal decision process**
that turns those facts into mutually responsive football and survives
competition.

The last chain makes that boundary unusually clear:

```text
DDD-0
  one defender's legal movement creates a real two-outlet dilemma        PASS

D-COVER-0
  shared intent changes the second defender's physical response          REAL
  endpoint-arrival handoff changes actual first control                  FAIL

D-LANE-0
  observer-grounded race along the complete intended pass path           VALID
  static corridor margin explains actual first control                   FAIL
```

The world responds to bodies, but neither a receiver endpoint nor a sampled
pass corridor tells a defender what an evolving group should do. Continuing to
add one more static fact would repeat the same failure pattern.

## 2. What the world model already has

### Live physical truth

* independent authoritative ball;
* oriented body access and screening;
* simultaneous loose-ball claims;
* contact before stable control;
* delayed/recontested control and third-player recovery;
* deterministic player and ball integration.

### Dormant or offline decision substrate

* observer-specific FOV, scanning, memory and deterministic observation error;
* current-model reach/turn estimates;
* generic off-ball candidate surfaces;
* relative moving-point feasibility;
* short-lived attacking and defensive movement commitments;
* generic team demand/claim occupancy types;
* five-class first-transition Oracle-v2;
* structural counterfactual clones.

These are not missing implementations. They are unqualified components whose
live causal composition has repeatedly failed or remains untested.

## 3. Remaining gaps, separated by kind

### A. Temporal multi-agent belief and responsibility — highest VISION value

The engine cannot yet represent a sequence such as:

```text
I start pressure
→ you observe my commitment and delay/cover
→ the carrier changes direction
→ I can no longer press
→ responsibility rotates without a commander reassigning everyone
```

Real attacking offers and defensive cover are negotiations through time. The
current dormant commitments are snapshots; the current TeamBrain remains a
commander. This gap explains missing check-and-go, weak-side adjustment,
step-and-cover, delay and rotating pressure more directly than another point
value does.

### B. Action transition and conditional value — highest inference value

The project still cannot reliably estimate from kick-time facts:

```text
P(intended / teammate / opponent / loose / dead)
× conditional outcome under each branch
```

Oracle-v2 fixed the event semantics. T0b and the factorised estimator banked
strong target-specific signal, but both stopped at their frozen calibration or
exact-conservation gates. Without this layer, a local choice may be physically
interesting but not knowably worth taking.

### C. Arrival/contact readiness — real physical gap, narrower visible payoff

Cheap ETA does not fully represent arrival velocity, braking, body orientation,
access side, screening and the delay from first touch to stable control. D-LANE
adds evidence that point ETA alone is not a transition model. A0 also showed
that relative speed by itself is not the missing universal mediator, so this
must be a coupled contact process rather than another scalar feature.

### D. Controlled possession between touches — real, high-risk local gap

B1d proved isolated fixed-rendezvous recovery. Live cadence, opponent claims and
possession continuity remain unproved. This improves ball feel, not the main
positioning/tactical violation, and remains parked behind the S3–S9 mainline.

### E. Secured shielding/tackle access — real but local

Loose-ball access reads oriented bodies; ordinary standing tackles still do
not. Q0 failed its first non-vacuity audit. This cannot be reopened as a generic
strength or shielding bonus.

### F. Low-priority rules/execution facts

Whole-ball-over-line, finer first touch, preferred foot, exact tackle cause and
richer duel vocabulary remain honest gaps. They are not blockers for emergent
team identity.

## 4. The decision fork

### Option A — temporal local micro-ecology (**recommended for VISION**)

Authorise one new, sealed, offline programme in which attacking and defending
players repeatedly perceive, commit, revoke and respond during live possession
windows. Selection is based on actual match competition, not reception rate,
endpoint coverage or an oracle-best label.

Required boundaries:

* generic directions/relations only; no named press, overlap, underlap, first,
  second or third defender action;
* every player reads its own `PerceptionSnapshot` plus explicit teammate intent;
* re-decision only at existing brain cadence or world events, not a fitted timer;
* carrier and ball stay live;
* heterogeneous probe-local preferences compete; no universal weight vector;
* actual goals/results are selection authority, with retention, progression,
  options and scramble reported as separate hard diagnostics;
* sealed seeds/population/generations; no production genes, saves or full league;
* default live baseline remains untouched;
* a pass only authorises a separately reviewed default-off visual sandbox.

Why it requires a user decision: D-COVER and D-LANE did not authorise ecology.
This option deliberately accepts a larger, riskier causal hypothesis because
the safer serial facts have reached diminishing returns.

### Option B — transition estimator re-entry

Commission a genuinely new inference programme on independent match clusters,
with an exact probability-composition authority and unseen-state calibration.
Do not patch T0b or increase factorisation iterations.

This is the cleanest epistemic route and the slowest route to visible football.
Even a pass would initially authorise only an offline estimator consumer audit.

### Option C — coupled arrival/contact process

Build one bounded foundation combining the exact player integrator, actual ball
ticks, oriented access, arrival velocity and delayed stable control. It must
explain both pass reception and interception on fresh counterfactuals; no single
new dimension may carry the claim.

This is safer than Option A and more directly physical, but it risks returning
to a long sequence of local audits while the commander and fixed formation stay
alive.

## 5. Recommendation

Choose **Option A** if the priority remains the VISION north star: visible,
emergent, competing team identities. The project has already learned that
single snapshots and global static weights cannot produce mutual adaptation.
The next meaningful hypothesis is the process itself.

The user selected Option A with an additional hard split: do not bundle the
closed process with competitive selection. The first authority is therefore
[`DECENTRALISED-DEFENSIVE-ROTATION-PROCESS.md`](DECENTRALISED-DEFENSIVE-ROTATION-PROCESS.md).
D-ROTATE-0 removes defensive mark/chaser publication in an isolated arm and
measures only whether local observation plus the existing live action cadence
produces a stable responsibility-rotation fingerprint. It contains no payoff,
preference, gene or evolution.

D-ROTATE-0 is now complete and failed without validity ambiguity. Local
responsibility facts were broadly observable (`3412/3440` eligible ticks) and
stable leaders existed in `34/64` states, but full rotation fingerprints appeared
in only `2/64` states against a frozen `16/64` gate. The existing local action
substrate therefore cannot replace commander assignments simply by knowing who
is closest. Selection and real-result filtering remain unopened.

The next causally distinct representation question is narrower than ecology:
can defenders publish and observe role-neutral movement intent/occupancy from
their own perception, so complementary pressure/cover relationships are
expressible without naming or allocating first/second/third defenders? That
language must be dormant and probe-only before any closed-loop consumer is
authorised.

The existing `DefensiveMovementCommitment` and C0 occupancy primitives already
cover the storage/accounting half of that language. The next frozen authority
therefore does not create another fact-only census. It is
[`DECENTRALISED-DEFENSIVE-INTENT-NEGOTIATION.md`](DECENTRALISED-DEFENSIVE-INTENT-NEGOTIATION.md):
a probe-only, world-event-triggered consumer in which defenders independently
settle duplicate opponent relations from their own perception and explicit
team-mate commitments, then execute only existing chase/mark/shape actions.
It remains a mechanism gate: no payoff, learning, results or ecology.

D-INTENT-0 is now complete and failed its primary mechanism gates. The local
consumer successfully formed multiple unique commitments in 64/64 states,
settled duplicate proposals in 64/64 and executed at least two defensive action
types in 61/64. Yet only 1/64 states produced the frozen temporal rotation
fingerprint. Decentralised allocation to opponent relations is therefore not
the missing temporal process.

This closes Option A's current defensive sub-branch before payoff or selection.
The project should not turn the same family into faster re-planning or add a
space/target score post hoc. The remaining already-ranked authorities are now
Option B (a new transition-estimator programme) or Option C (a coupled
arrival/contact process), unless a future contract supplies a genuinely new
multi-body temporal state rather than another local relation selector.

Choose **Option B** only if trustworthy pass choice must precede visible
positioning. Choose **Option C** only if physical first-control realism now
outranks tactical emergence.

Until that decision, the accepted authority is:

```text
live B0 + existing commander
D-COVER/D-LANE/D-ROTATE/D-INTENT parked
no defensive sandbox
no ecology
no new selector
```
